## Obligations

Systems are expected to handle specific data elements according to the role (actor) they play in the EHDS Imaging Report ecosystem. In this specification, these expectations are defined as [FHIR Implementation Obligations](https://hl7.org/fhir/R5/obligations.html), following [EHDS shared practices](https://build.fhir.org/ig/Xt-EHR/xt-ehr-common/branches/new-pages/obligations.html) and are listed in the [Artifacts](artifacts-obligations.html) section.

At present, these obligations are defined directly in this IG. In the future, they are expected to be defined in the Xt-EHR logical models and inherited by this IG’s resources and elements.
